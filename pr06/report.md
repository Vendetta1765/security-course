# ПР №6. Анализ сетевого трафика (tcpdump / Wireshark)

**Дисциплина:** Программно-аппаратная защита информации
**Тема:** Мониторинг и анализ сетевого трафика. Перехват данных в открытом виде.

---

## 1. Базовый захват трафика (tcpdump)

Утилита `tcpdump` — консольный анализатор сетевых пакетов. Список доступных интерфейсов получен командой `tcpdump -D`, активный интерфейс — `enp0s3`.

Выполнен захват пакетов командой `tcpdump -i enp0s3 -c 10` (ограничение в 10 пакетов).

Назначение ключей:
- `-i <интерфейс>` — на каком интерфейсе слушать трафик;
- `-c <N>` — захватить ровно N пакетов и остановиться;
- `-nn` — не преобразовывать IP-адреса в имена и номера портов в названия служб;
- `-vvv` — максимально подробный вывод;
- `-X` — показать содержимое пакета в шестнадцатеричном виде и ASCII;
- `-A` — показать содержимое пакета только в ASCII.

---

## 2. Фильтрация трафика

Применены фильтры захвата (BPF):

| Фильтр | Что отбирает |
|--------|--------------|
| `port 53` | DNS-запросы и ответы |
| `port 80` | HTTP-трафик (открытый) |
| `port 443` | HTTPS-трафик (зашифрованный) |
| `host 8.8.8.8` | Весь трафик с конкретным узлом |
| `icmp` | ICMP-пакеты (ping) |

### Ключевой эксперимент: HTTP vs HTTPS

При захвате трафика на порту 80 (`tcpdump -A port 80`) во время запроса `curl http://example.com` в перехваченных пакетах видно содержимое в открытом виде: строки `GET / HTTP/1.1`, `Host: example.com`, заголовки и тело ответа читаются как обычный текст.

При захвате на порту 443 во время запроса `curl https://example.com` содержимое пакетов нечитаемо — передаётся зашифрованный шифртекст TLS.

**Вывод:** протокол HTTP передаёт данные (включая логины и пароли) в открытом виде, и любой, кто имеет доступ к сети, может их перехватить. HTTPS шифрует трафик, защищая его от перехвата.

---

## 3. Сохранение и анализ дампа

Трафик сохранён в файл командой `tcpdump -i enp0s3 -w capture.pcap -c 50`.
Чтение из файла — `tcpdump -r capture.pcap`.
Статистика по протоколам получена через `tshark -r capture.pcap -q -z io,phs`.

Формат `.pcap` — стандартный формат хранения дампов трафика, читается tcpdump, tshark и Wireshark.

---

## 4. Анализ в Wireshark (GUI)

Файл `capture.pcap` открыт в Wireshark. Использованы фильтры отображения (display filters):

| Фильтр | Назначение |
|--------|------------|
| `http` | Только HTTP-пакеты |
| `dns` | Только DNS-запросы/ответы |
| `ip.addr == 8.8.8.8` | Трафик с конкретным IP |
| `tcp.port == 443` | HTTPS-соединения |
| `icmp` | ICMP (ping) |

Функция **Follow → HTTP Stream** позволяет собрать из отдельных пакетов целый диалог «запрос-ответ» и увидеть его в читаемом виде — наглядно демонстрирует уязвимость открытого протокола.

**Разница между фильтрами захвата и отображения:** фильтр захвата (синтаксис BPF, задаётся при запуске) отсеивает пакеты ещё до записи — они вообще не попадают в дамп. Фильтр отображения (синтаксис Wireshark) применяется к уже захваченным пакетам и лишь скрывает лишнее из вида, не удаляя данные.

## Ответы на контрольные вопросы

1. **Чем опасен перехват трафика:** злоумышленник в той же сети (например, в открытом Wi-Fi) может прочитать передаваемые в открытом виде данные — логины, пароли, cookie. Это сниффинг / атака «человек посередине» (MITM).
2. **Почему HTTP небезопасен, а HTTPS — да:** HTTP передаёт данные открытым текстом, HTTPS шифрует их по TLS, поэтому перехваченный трафик нечитаем без ключей.
3. **Можно ли расшифровать перехваченный HTTPS:** в общем случае нет — без приватного ключа сервера или сессионных ключей.
4. **Зачем нужен promiscuous mode:** режим, в котором сетевая карта принимает все пакеты сегмента, а не только адресованные ей. Нужен для прослушивания чужого трафика.
5. **Почему для захвата нужны права root / группа wireshark:** доступ к «сырому» сетевому интерфейсу — привилегированная операция, иначе любой пользователь мог бы прослушивать трафик системы.
6. **Как защититься от перехвата:** использовать HTTPS/TLS, VPN, шифрование на уровне приложений, избегать открытых протоколов (HTTP, Telnet, FTP).
7. **Легальность:** перехват чужого трафика без разрешения — нарушение закона. Анализ допустим только в своей сети или с письменного согласия владельца.

## Выводы

В ходе работы освоены инструменты анализа сетевого трафика — консольный `tcpdump`/`tshark` и графический Wireshark. Изучены ключи захвата и синтаксис фильтров (захвата и отображения).
